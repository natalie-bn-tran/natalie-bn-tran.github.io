Note: For the Linux machines, all commands were run as the root user.

Network Configuration

Completing the networking for each of the machines is crucial. This will give the machines their own IP address, allowing communication, data transportation, and sets the stage for changing network settings.
IP Address: A unique identifier for a device on a network.

To make things easier, the LAN subnet will be same as the one automatically set in the pfSense router (192.168.1.0/24), and the last octets of the non-router machines will be 2-10.

pfSense

WAN: It's set with DHCP, but it's recommended to set it statically.
LAN: This can be kept to the default IP address, but can be changed as well.

Change the IP address of a pfSense router, using the CLI:

  1. Enter 2
  2. Enter 1 or 2, depending on if the WAN or LAN is being changed
  3. Enter the new IP address in this format: xxx.xxx.xxx.xxx/yy
    • Note: xxx.xxx.xxx.xxx is the IP address, and /yy is the subnet in CIDR format (the subnet can be entered after if it was not written with the IP address).
    • For WAN: yy = 16
    • For LAN: yy = 24
  4. For WAN: the default gateway is 172.16.0.1, and enter y when prompted if this should be the default upstream gateway
    • Note: This only applies for if you're using the SDC infrastructure. 172.16.0.1 is the IP address of the SDC's router and will allow the VMs to have Internet access.
  5. Enter n or hit enter for the rest of the options

For the lab, it is recommended to have 1:1 NAT set up on the router to have the ability to work from the host machine.
Network Address Translation (NAT): Translates IP addresses between interfaces.
1:1 NAT: Translates a WAN IP address to a LAN IP address and vice versa.

  • Having 1:1 NAT grants the ability to access the various services on a host machine instead of the VMs.
  • Ex: 172.16.187.0/24 would translate to 192.168.1.0/24, so 172.16.187.23 is 192.168.1.23.

Set up 1:1 NAT:

  1. On the host machine, open a browser and enter the pfSense's WAN IP address
  2. Login with the credentials admin : pfsense
  3. Create a 1:1 NAT rule:
    1. Navigate to Firewall > NAT > 1:1
    2. Change External Subnet IP to WAN Address
    3. Change Internal IP to LAN net
    4. Save and apply the changes
  4. Configure a virtual IP:
    1. Navigate to Firewall > Virtual IP
    2. IP type: Proxy ARP
    3. Interface: WAN
    4. Address type: Network
    5. Address(es): 172.16.xxx.0/24

      6. Note: Here, the subnet is /24 and not /16 because the only octets that are translated are the first three (network ID), and the last octet (host ID) needs to remain the same across the translation. /16 is for setting up the IP and making sure it's on the same subnet as the SDC router.

    6. Save and apply the changes

    Note: Replace 241 with xxx.

Windows

The network configuration is the same for all of the Windows machines.

Change the IP address of a Windows machine:

  1. Run Win + R and enter ncpa.cpl or navigate to Control Panel > Network and Internet > Network and Sharing Center
  2. Select Ethernet > Properties > IPv4 > Properties
  3. Select Use the following IP Address and fill out the information:
    1. IP Address: 192.168.1.x
    2. Subnet Mask: 255.255.255.0 (automatically fills out when clicked)
    3. Default Gateway: 192.168.1.1 (router's LAN IP address)
    4. Preferred DNS server : 8.8.8.8 (Google's Public DNS)
  4. Select OK
Linux

Unlike Windows, the Linux machines have their IPs configured differently, although there are some similarities between the configurations.

Note: XX in ensXX is the interface number, and can be found by running ip a.

Ubuntu 22.04

Change the IP:

  1. Edit the /etc/netplan/xxx-network-manager-all.yaml file

    2. Note: xxx is a number.

  2. Change the configuration to the following:
    3. network:
    version: 2
        renderer: networkd
        ethernets:
            ensXX:
                addresses:
                  - 192.168.1.7/24
                routes:
                  - to: default
                    via: 192.168.1.1
                nameservers:
                  addresses: [8.8.8.8]
  3. Edit the /etc/netplan/xx-installer-config.yaml file and change DHCP4 to false
  4. Run one of the following commands to apply the new configuration:
    5. netplan apply
systemctl restart systemd-networkd
CentOS 7

Change the IP:

  1. Edit the /etc/sysconfig/network-scripts/ifcfg-ensXX file
  2. Change the configuration to the following:
    3. TYPE=Ethernet
PROXY_METHOD=none
BROWSER_ONLY=no
BOOTPROTO=static
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_FAILURE_FATAL=no
IPV6_ADDR_GEN_MODE=stable-privacy
NAME=ensXXX
UUID=20917e75-ac6f-42c1-8a1c-cb0af2f01a9b
DEVICE=ensXXX
ONBOOT=yes
IPADDR=192.168.1.8
NETMASK=255.255.255.0
GATEWAY=192.168.1.1
  3. Run the following command to apply the new configuration:
    4. systemctl restart network
Debian 11

Change the IP:

  1. Edit the /etc/network/interfaces file
  2. Change the configuration to the following:
    3. auto lo
iface lo inet loopback

auto ensXX
iface ensXX inet static
    address 192.168.1.9
    netmask 255.255.255.0
    gateway 192.168.1.1
  3. Run the following command to apply the new configuration:
    4. systemctl restart network
Ubuntu 18

Change the IP:

  1. Edit the /etc/netplan/xx-network-manager-all.yaml file
  2. Change the configuration to the following:
    3. network:
    version: 2
        renderer: networkd
        ethernets:
            ensXX:
                addresses:
                  - 192.168.1.10/24
                nameservers:
                  addresses: [8.8.8.8]
                gateway4: 192.168.1.1
  3. Run the following command to apply the new configuration:
    4. systemctl restart network

Active Directory

Active Directory (AD): A giant database and central management for domain users, computers, and services.
There are two main components of AD: LDAP and DNS

  • LDAP: Lightweight Directory Access Protocol - it puts the directory in active directory.
  • DNS: Domain Name System - allows LDAP to reference names in its directory.

In Evan's Lab:

  • Windows Server 2022: Primary domain controller (DC) and DNS server
  • Windows Server 2012 R2: Secondary DC and DNS server
Domain Controllers

There are two parts of setting up domain controllers: installing Active Directory Domain Services, which is the same for both DCs, and promoting the device to a DC.

Windows Server 2022: Primary DC

DNS settings: change the DNS to 127.0.0.1 (localhost)
Localhost: the hostname of the machine (refers to itself).

Install AD:

  1. Navigate to Server Manager > Manage > Add Roles and Features or
    Control Panel > Programs > Programs and Features > Turn Windows Features On or Off
  2. Make the following selections and click next for other options:
    1. Installation Type: Role-Based
    2. Server Roles: Active Directory Domain Services
  3. Install
  4. Rename the device using PowerShell:
    5. Rename-Computer -NewName [name] -restart

    Note: The machines in the network will be given names with the form: SERVICE0X.
    Ex: The DCs will be named DC01 and DC02, respectively.

Create a primary DC:

  1. Navigate to Server Manager > Flag > Promote to Domain Controller
  2. Make the following selections for the pop-up and click next for other options:
    1. Deployment Configuration: add a new forest and enter a domain name

      2. Note: Adding a new forest creates a new domain.

    2. Domain name: Enter a FQDN

      3. FQDN: Fully Qualified Domain Name
      Ex: ejuan.detergent

    3. Domain Controller Options: Set the functional level and password

      4. Functional Level: Oldest version of Windows that any domain controller has; in this case, it would be Windows Server 2012 R2.

    4. Password: To make things easier, the password will be the same as the one set for the device
    5. Additional Options: ensure the domain is correct; it's one capitalized word

      6. Ex: EJUAN

    6. Prerequisites Check: Install
  3. Wait for the machine to restart
Domain Joining

To join the other machines to the AD, the DNS settings must point to the DNS servers, allowing the Domain to be discovered and joined.

Changing DNS Settings

The DNS settings must be changed so that the primary DNS points to the primary DC's IP, and the secondary DNS points to the secondary DC's IP.

Windows
Set the DNS the same way the IP addresses were configured.

Ubuntu 22 & 18
Set the DNS settings with the network file:

  1. In the /etc/netplan/xx-network-manager-all.yaml file replace 8.8.8.8: 
    2. nameservers:
    addresses: [192.168.1.2,192.168.1.4]
  2. Apply the network changes:
    3. netplan apply

CentOS 7
Set the DNS settings with the network file:

  1. Add to the /etc/sysconfig/network-scripts/ifcfg-ensXX file:
    2. DNS1=[192.168.1.2]
DNS2=[192.168.1.4]
  2. Apply the network changes:
    3. systemctl restart network

Debian 11
Set the DNS settings with the GUI:

  1. Navigate to Settings > Network > Wired Settings > IPv4
  2. In the DNS section, deselect Automatic and enter the DNS servers' IPs
  3. Apply the changes by toggling the wired connection
Joining the Domain

Domain joining is fairly simple. The machine's hostname will be changed as good practice, so the machine can be easily identified on the network, then it will find and join the domain.

Windows 7
Rename the computer and add to the domain:

  1. Navigate to Start > Run > sysdm.cpl > Computer Name > Change
    1. Change the computer's name
    2. Member of: Domain
    3. Enter the FQDN
    4. Enter the administrator credentials
  2. Wait for the restart

Other Windows
Rename the computer and add to the domain in PowerShell:

  1. Rename the computer:
    1. Run the following command:
      2. Rename-Computer -NewName [name] -restart
  2. Add the computer to the domain:
    1. Run the following command:
      2. Add-Computer -DomainName [FQDN] -Restart
    2. Enter the domain administrator credentials
  3. Wait for the restart

Ubuntu 22
Rename the computer:

  1. Run the following command:
    2. hostnamectl set-hostname [name]
  2. Run exit until the login screen and login again

Add the computer to the domain:
Note: This is the same for all Linux machines, but the required packages are different.

  1. Discover the domain:
    2. realm discover [FQDN]

    Note: The realmd package will need to be installed if it isn't already.

  2. Join the domain:
    3. realm join [FQDN]
  3. Edit the /etc/pam.d/common-session file and add the following lines:
    4. session optional pam_mkhomedir.so skel=/etc/skel umask=077
  4. Authenticate to check if the machine was machine joined by signing in as Administrator@[FQDN]

Ubuntu 18 and CentOS 7
Rename the computer:

  1. Run the following command:
    2. hostnamectl set-hostname [new name]
  2. Edit the /etc/hosts file, and replace the old hostname with the new one
  3. Reboot the machine:
    4. reboot

Add the computer to the domain:

  1. Search, realm join , and authenticate like in Ubuntu 22

    2. Note: The realmd, packagekit (Ubuntu 22), and PackageKit (CentOS 7) packages will need to be installed if they aren't already.

Debian 11
Rename the computer like with Ubuntu 18.

Add the computer to the domain:

  1. Edit the /etc/resolv.conf file:
    2. domain [FQDN]
search [FQDN]
  2. Search and realm join like in Ubuntu 22

    3. Note: The realmd and sssd packages will need to be installed if they aren't already.

  3. Edit the `/etc/pam.d/common-session` file if needed by adding:
    4. session optional        pam_mkhomedir.so skel=/etc/skel umask=077
  4. Authenticate

MySQL

Machine: Ubuntu 22
Purpose: MySQL database for XAMPP and Apache
Note: mysql_native_password is now deprecated.

Install MySQL:

  1. Update and upgrade:
    2. apt update && apt upgrade -y
  2. Install the MySQL server and validate the installation:
    3. apt install mysql-server -y
mysqld --version
  3. Check the status of the service and start if needed:
    4. service mysql status
service mysql start
  4. Run the MySQL secure installation and answer the prompts as such:
    5. mysql_secure_installation

validate password strength: y
levels: 2
change root password: y (can use same password)
continue with password: y
remove anonymous use: y
disallow root login remotely: y
remove test database: y
reload privilege tables: y
  5. Restart MySQL:
    6. service mysql restart
  6. Login to the MySQL server:
    7. mysql -u root -p

Allow remote connections:

  1. Edit the /etc/mysql/mysqld.cnf file:
    2. bind-address          0.0.0.0

Each website needs their own database and database user so the websites are able to have their data isolated. Separated users adds a layer of security, allowing further separation between the websites.

Set up users and databases:

  1. Create user:
    2. CREATE USER '[user]'@'%' IDENTIFIED WITH mysql_native_password BY '[password]';
flush privileges;
  2. Create database:
    3. create database [database];
  3. Grant database privileges to user:
    4. grant all privileges on [database].* to '[user]'@'%';
  4. Repeat for another user
